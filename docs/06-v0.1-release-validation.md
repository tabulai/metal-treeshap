# v0.1.0 release validation

This document records the local release-candidate validation performed on 2026-07-12. CI,
TestPyPI, and PyPI results are separate evidence: they must be produced by the checked-in
workflows after the GitHub repository and trusted publishers exist.

## Material Passport

- Origin skill: `academic-research-suite/experiment-agent`
- Origin mode: `run + validate`
- Machine: Apple M4 Max (40-core GPU), Mac16,6
- OS: macOS 26.2 (25C56)
- Verification status: `PARTIAL` until hosted CI, TestPyPI, PyPI, and privileged power gates run
- Version label: `v0.1.0-rc-local`

## Correctness and compatibility

- Fresh Release CMake build: successful with runtime Metal shader compilation.
- Complete CTest suite: 17/17 passed, including all eight fixtures through atomic,
  SIMD-group, deterministic Kahan, and tiled-atomic paths.
- Focused extractor validation: 7/7 malformed/layout tests passed.
- Phase 2.1 tool validation: 13/13 passed.
- Source-only package path: missing-native diagnostic and internal-import propagation passed.
- XGBoost golden suites:
  - 2.0.3: all 16 applicable cases plus unsupported-objective rejection passed.
  - 3.1.2: all 16 applicable cases plus unsupported-objective rejection passed.
  - 3.3.0: all 17 cases plus unsupported-objective rejection passed.
- New XGBoost 3.3 flattened-DART fixture: maximum reference attribution error
  `1.626e-7`, sum-to-margin error `1.520e-7`, and Metal error `3.576e-7`.
- An independent four-class XGBoost 3.3 direct-dropout adversarial model also passed, with
  `9.14e-8` maximum attribution error.

## Distribution matrix

Native macOS ARM64 wheels were built and installed for CPython 3.10, 3.11, 3.12, 3.13,
and 3.14. Each wheel passed all five installed public-API tests on the M4 Max. The source
distribution and wheels passed metadata/content validation and `twine check`; required
shader, extractor, typing marker, and native extension files were present, while Python
cache artifacts were absent.

The GitHub workflows additionally enforce portable Linux builds, the three-version XGBoost
matrix, an explicit Metal-device probe, all on-device CTests, release tag/version equality,
artifact hashes, trusted publishing, and clean TestPyPI/PyPI install tests. `actionlint
1.7.12` reports no workflow errors.

## Performance and power boundary

The release preserves the Phase 2.1 production defaults and measured throughput results;
this compatibility/packaging round did not retune them. The controlled `powermetrics` path
is implemented, but local non-interactive `sudo` is unavailable. Consequently no energy or
performance-per-watt claim is made and that privileged gate remains unverified.

## Fallacy scan

All eleven experiment-agent categories were checked. Simpson's paradox, ecological inference,
Berkson selection, collider adjustment, base-rate neglect, regression to the mean,
survivorship bias, correlation/causation, and reverse causality do not apply to these direct
executable compatibility checks. Look-elsewhere and garden-of-forking-paths risks remain
relevant to the earlier performance tuning; the release does not reinterpret those exploratory
results, and production choices remain based on the separately blocked finalist experiments.
